# 智能家居控制系统的设计与实现

## 摘要

随着物联网技术的快速发展，智能家居系统逐渐成为现代生活的重要组成部分。本文设计并实现了一套基于OneNET物联网平台的智能家居控制系统，采用Django上位机与STM32下位机的协同架构。上位机通过HTTP轮询方式从OneNET平台获取设备物模型数据，实现传感器数据的实时展示、设备远程控制以及基于机器学习的温度预测功能；下位机采用STM32F103C8T6微控制器，通过ESP8266 WiFi模块与OneNET平台建立MQTT连接，实现温湿度、光照等传感器数据的采集与上报，以及风扇调速、LED自动控制等执行器的驱动。系统经过完整测试，各项功能运行稳定，达到了预期设计目标。

**关键词：** 智能家居；OneNET物联网平台；Django；STM32；MQTT；机器学习

## Abstract

With the rapid development of Internet of Things (IoT) technology, smart home systems have gradually become an important part of modern life. This paper designs and implements a smart home control system based on the OneNET IoT platform, using a collaborative architecture of Django upper computer and STM32 lower computer. The upper computer obtains device thing model data from the OneNET platform through HTTP polling, realizing real-time display of sensor data, remote device control, and temperature prediction based on machine learning. The lower computer uses an STM32F103C8T6 microcontroller, establishing an MQTT connection with the OneNET platform through an ESP8266 WiFi module, realizing the collection and reporting of temperature, humidity, light and other sensor data, as well as the driving of actuators such as fan speed control and LED automatic control. After comprehensive testing, all functions of the system operate stably and have achieved the expected design goals.

**Key words:** Smart Home; OneNET IoT Platform; Django; STM32; MQTT; Machine Learning

## 一、需求分析

### 1.1 课题背景

随着物联网（IoT）技术的迅猛发展，智能家居已经从概念走向现实，成为提升生活品质的重要手段。传统的家居控制方式依赖人工操作，无法实现远程监控和智能化管理。基于物联网平台的智能家居系统能够将各类传感器、执行器通过网络连接，实现数据的远程采集、传输、存储和分析，为用户提供便捷、智能的家居管理体验。

中国移动OneNET平台是国内主流的物联网开放平台之一，提供了完善的设备接入、数据存储、规则引擎等服务，支持MQTT、HTTP等多种协议，适合作为智能家居系统的云端服务平台。

### 1.2 问题描述

本课题需要解决以下核心问题：

（1）如何实现下位机（STM32）与云平台（OneNET）之间的稳定通信，确保传感器数据的可靠上报和控制指令的准确下发；

（2）如何设计上位机（Django）系统，通过HTTP接口从OneNET平台轮询获取设备数据，并提供友好的人机交互界面；

（3）如何利用机器学习算法对历史温度数据进行分析和预测，实现风扇的智能化控制；

（4）如何实现LED照明的自动控制，根据环境光照强度自动调节灯光状态。

### 1.3 功能需求

根据课题要求，系统需要实现以下功能：

（1）**传感器数据采集**：采集温度（DHT11）、光照强度（光敏电阻）数据；

（2）**数据远程上报**：通过MQTT协议将传感器数据上报至OneNET物联网平台；

（3）**设备远程控制**：通过OneNET平台下发控制指令，实现风扇调速（0-1000 r/min，PWM控制）；

（4）**LED自动控制**：当光照强度低于1000 Lux时自动点亮LED灯；

（5）**Web实时监控**：Django上位机提供Web界面，实时显示传感器数据和设备状态；

（6）**温度预测**：基于机器学习算法（随机森林回归）预测未来温度，提供风扇转速建议；

（7）**自动控制模式**：根据预测温度自动调节风扇转速。

### 1.4 性能需求

（1）数据轮询间隔：3秒；

（2）控制指令响应时间：不超过10秒；

（3）系统稳定性：支持长时间连续运行；

（4）数据存储：保留最近500条历史记录。

- 设置回复：`$sys/{product_id}/{device_name}/thing/property/set_reply`

### 2.5 遇到的问题与解决方法

在系统设计过程中，遇到了以下主要问题：

**问题一：OneNET API Token认证失败**

下位机使用的MQTT Token与HTTP API所需的Token格式不同。MQTT Token的资源标识为设备级（`res=products/.../devices/...`），而HTTP API需要产品级Token（`res=products/...`）。通过查阅OneNET官方文档，采用正确的Token生成算法解决了此问题。

**问题二：MQTT控制指令丢失**

下位机在通过MQTT上报数据时，`MQTT_Publish`函数会清空UART接收缓冲区，导致同时到达的控制指令被丢弃。解决方法是在发送前后都检查缓冲区中是否有待处理的数据。

**问题三：OneNET下发JSON格式差异**

OneNET平台的属性上报（post）和属性设置（set）使用不同的JSON格式。上报格式为`"key":{"value":val}`，而下发格式为`"key":val`。下位机的JSON解析代码需要匹配正确的格式才能正确提取控制参数。

#### 3.1.2 关键代码说明

**风扇PWM控制函数：**

```c
void ControlFan(int16_t speed)
{
    fan_speed = speed;
    if (speed == 0) {
        HAL_TIM_PWM_Stop(&htim4, TIM_CHANNEL_1);
        HAL_GPIO_WritePin(IN1_GPIO_Port, IN1_Pin, GPIO_PIN_RESET);
    } else {
        HAL_TIM_PWM_Start(&htim4, TIM_CHANNEL_1);
        uint32_t pulse = (uint32_t)speed * htim4.Init.Period / 1000;
        __HAL_TIM_SET_COMPARE(&htim4, TIM_CHANNEL_1, pulse);
    }
}
```

该函数实现了风扇转速的精确控制。当转速为0时，停止PWM输出并强制拉低GPIO引脚，确保电机完全停止；当转速大于0时，启动PWM并根据转速值计算占空比。TIM4的预分频器设置为72-1，自动重载值为1000-1，产生1kHz的PWM信号。

**LED自动控制函数：**

```c
void ControlLedByLight(void)
{
    if (light_value < 1000) {
        HAL_GPIO_WritePin(IN1_GPIO_Port, IN1_Pin, GPIO_PIN_RESET);
        led_on = 1;
    } else {
        HAL_GPIO_WritePin(IN1_GPIO_Port, IN1_Pin, GPIO_PIN_SET);
        led_on = 0;
    }
}
```

该函数根据光照强度自动控制LED灯。由于LED电路采用低电平点亮设计，当光照低于1000 Lux时输出低电平点亮LED，否则输出高电平熄灭LED。

### 3.3 机器学习预测模块

#### 3.3.1 算法选择

本系统采用随机森林回归（RandomForestRegressor）算法进行温度预测。选择该算法的原因：

（1）随机森林对小规模数据集具有良好的泛化能力；

（2）能够捕捉非线性关系，适合温度变化的复杂模式；

（3）不需要复杂的特征工程，对异常值具有鲁棒性。

#### 3.3.2 特征工程

采用滑动窗口方式构建特征：

```python
def _build_features(temps):
    X, y = [], []
    for i in range(WINDOW_SIZE, len(temps)):
        window = temps[i - WINDOW_SIZE:i]
        idx = i % 60
        features = window + [idx // 6, idx % 6 * 10]
        X.append(features)
        y.append(temps[i])
    return np.array(X), np.array(y)
```

每条样本包含5个历史温度值和2个时间特征（小时、分钟），目标值为下一时刻的温度。

#### 3.3.3 风扇控制策略

根据预测温度自动计算建议风扇转速：

**表3.5  温度-风扇映射表**

| 预测温度 | 风扇转速 | 说明 |
|---------|---------|------|
| ≥ 32°C | 1000 r/min | 全速运转 |

## 五、结论与心得

### 5.1 设计总结

本课题成功设计并实现了一套完整的智能家居控制系统，主要完成了以下工作：

（1）完成了STM32下位机的硬件设计和软件开发，实现了DHT11温度传感器、光照传感器的数据采集，以及风扇PWM调速、LED自动控制等执行器的驱动；

（2）完成了ESP8266 WiFi模块的MQTT通信对接，实现了与OneNET物联网平台的稳定连接和数据交互；

（3）完成了Django上位机的开发，实现了OneNET HTTP API的封装、传感器数据的轮询展示、设备远程控制等功能；

（4）实现了基于随机森林回归的温度预测功能，能够根据历史数据预测未来温度并自动调节风扇转速；

（5）解决了开发过程中遇到的多个技术问题，包括Token认证、MQTT消息丢失、JSON格式差异、PWM控制精度等。

### 5.2 遇到的困难与解决办法

**困难一：OneNET平台API调试**

OneNET平台的HTTP API和MQTT协议在Token认证、消息格式等方面存在差异，且官方文档描述不够详细。通过反复测试和查阅社区资料，逐步理清了各种Token的用途和生成方法。

**困难二：MQTT消息可靠性**

ESP8266在发送MQTT消息时会阻塞UART接收，导致同时到达的控制指令丢失。通过在发送前后增加缓冲区检查机制，并添加定期重新订阅功能，提高了消息接收的可靠性。

**困难三：前端状态同步**

由于OneNET API存在网络延迟，前端按钮的高亮状态容易被轮询数据覆盖。通过引入冷却期机制和请求序号防竞争机制，解决了状态不同步的问题。

### 5.3 心得体会

通过本次课程设计，我深入学习了物联网系统的完整开发流程，从硬件选型、嵌入式编程、云平台对接到Web应用开发，涵盖了多个技术领域的知识。

在嵌入式开发方面，我掌握了STM32的GPIO、UART、TIM、ADC等外设的配置和使用，学会了通过AT指令控制ESP8266 WiFi模块，理解了MQTT协议的工作原理。

在Web开发方面，我学会了使用Django框架构建RESTful API，了解了前后端分离的开发模式，以及如何通过JavaScript实现页面的实时更新。

在机器学习方面，我了解了随机森林回归算法的基本原理，学会了如何将机器学习模型集成到实际应用中，实现智能化的数据分析和预测。

最重要的是，我学会了如何分析和解决开发过程中遇到的各种问题，培养了独立思考和动手实践的能力。这些经验将对我今后的学习和工作产生积极的影响。

### 5.4 未来展望

由于时间限制，本系统还有一些可以改进的地方：

（1）增加更多传感器类型，如烟雾检测、人体红外感应等，扩展系统的安防功能；

（2）引入深度学习算法（如LSTM），提高温度预测的精度；
